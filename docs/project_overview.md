**(2) 팀이름 + 챔프 총 12개의 컬럼을 OneHotEncoder 로 인코딩해서 숫자 벡터로 바꿈** 

* OneHotEncoding 이란?: 대부분의 머신러닝 모델은 문자를 처리할 수 없고, 숫자 벡터만 입력으로 받을 수 있음 (ex: 입력으로 zed 는 불가능), 그렇기 때문에 챔피언, 팀을 0, 1 로 만 이루어진 벡터로 바꿔주는 작업, 각 챔피언/팀을 독립적인 차원으로 표현하기 때문에 쓸데없는 정보로 오차가 생기지 않는다.



* 어디에서 import 했는지: scikit-learn (싸이킷 런) - 머신러닝 알고리즘/ 전처리 도구가 들어있는 라이브러리다, OneHotEncoder 뿐 아니라 과제에서 사용한 로지스틱 회귀 모델, train/test 분할 같은 기능들을 제공함

---

**(3) 인코딩된 피처 (앞선 12개의 컬럼)를 입력, 블루팀의 승리 여부를 타깃으로 로지스틱 회귀 이진분류 모델을 학습한다** 

* 로지스틱 회귀 이진 분류 모델이란?: 벡터화 된 입력들을 분석해서 결과가 '두가지 중 어느쪽일지' 에 대한 확률 (0~1)을 계산해주는 모델이다.



* 모델 구조: 사용 알고리즘: scikit-learn의 LogisticRegression

&nbsp;	하는일: 벡터에 피처별 가중치를 곱해 점수 z를 만든 뒤, 함수를 적용해 0~1 사이 값으로 바꾼다. -> 이 값은 블루 팀이 이길 확률로 해석됨



* 모델 선택 이유: 승/패 처럼 두 가지인 이진 분류 문제에 가장 기본적으로 많이 쓰이는 모델이고 출력이 확률이기 때문에 UI에서 보여주기 좋기 때문

---

**(4) train/test 나눠서 정확도 출력, 픽만 썼을때 성능 확인**

* train/test 나눈다는것의 의미?:

&nbsp;       전체 데이터를 학습용/검증용 두 덩어리로 쪼갬

* 두 덩어리로쪼개는 이유: 같은 데이터로 학습하고, 같은 데이터로 평가하면모델이 그 데이터만 외워도 점수가 높게 나오기 때문, 과제에선 train/test 를 8:2 로 배분함

**(5) 앞서 사용한 OneHotEncoder 를 pick\_encoder.joblib 에 저장** 

* 저장하는 이유?: 모델뿐 아니라 인코더도 팀/챔프 카테고리 등을 학습 하기 때문, 사용자에게 팀, 챔피언을 입력받고 같은 규칙, 순서로 인코딩함.
* .joblib 확장자란?: 파이썬 객체 (특히 머신러닝 모델)를 빠르게 저장/불러오기 위한 라이브러리 과제에선 인코더/모델 둘다 .joblib 형식으로 저장했음.
